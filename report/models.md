# Обоснование выбора LLM для агентов

## Выбранная модель

**Qwen 3 Coder Next** (`qwen/qwen3-coder-next`) через API vsellm.ru (OpenAI-совместимый endpoint).

## Почему Qwen 3 Coder Next

### 1. Специализация на генерации кода

Qwen 3 Coder Next — модель, оптимизированная для задач кодогенерации. В мультиагентной системе каждый агент (DataAgent, ModelAgent) генерирует Python-код, который исполняется напрямую. Качество генерации кода критически важно для корректной работы всей системы.

### 2. Open source

Модель полностью открытая (Apache 2.0), что соответствует требованию проекта об использовании open source моделей.

### 3. Длинный контекст

Поддержка большого контекстного окна позволяет передавать в промпт профиль данных, документацию библиотек из RAG, историю итераций и обратную связь — всё одновременно, без потери информации.

### 4. Стоимость

Доступ через vsellm.ru значительно дешевле коммерческих API (GPT-4o, Claude). Это позволяет запускать систему многократно для экспериментов и бенчмаркинга без значительных затрат.

## Настройки температуры

Температура подобрана под задачу каждого агента:

| Агент | Temperature | Обоснование |
|-------|-------------|-------------|
| DataAgent | 0.3 | Предобработка данных требует детерминированного, надежного кода. Низкая температура снижает вероятность ошибок. |
| ModelAgent | 0.5 | Обучение моделей допускает больше вариативности: выбор гиперпараметров, архитектуры модели. Умеренная температура поощряет разнообразие решений. |
| EvalAgent | 0.3 | Анализ метрик и генерация обратной связи должны быть точными и конкретными. |
| Orchestrator | 0.5 | Оркестратор планирует стратегию и принимает решения. Умеренная температура позволяет находить нестандартные подходы при итеративном улучшении. |

## Единая модель для всех агентов

Все агенты используют одну модель. Это решение обосновано:
- **Простота**: один API endpoint, одна конфигурация
- **Достаточность**: Qwen 3 Coder Next справляется со всеми задачами системы (генерация кода, анализ метрик, планирование)
- **Воспроизводимость**: результат зависит только от одной модели, что упрощает бенчмаркинг
